# Outcomes

As the name suggests, outcomes are a core part of Protocol. On this page, we'll dive into the different outcome endpoints you can use to manage outcomes programmatically. We'll look at how to query, create, update, and delete outcomes.

## The outcome model

The outcome model contains all the information about your outcomes, such as their description.

### Properties

<Properties>
  <Property name="outcomes_id" type="string">
    Unique identifier for the outcome.
  </Property>
  <Property name="description" type="string">
    The description for the outcome.
  </Property>
</Properties>

---

## List all outcomes {{ tag: 'GET', label: '/v1/outcomes' }}

<Row>
  <Col>

    This endpoint allows you to retrieve a paginated list of all your outcomes. By default, a maximum of ten outcomes are shown per page.

    ### Optional attributes

    <Properties>
      <Property name="limit" type="integer">
        Limit the number of outcomes returned.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/v1/outcomes">

    ```bash {{ title: 'cURL' }}
    curl -G https://api.protocol.chat/v1/outcomes \
      -H "Authorization: Bearer {token}" \
      -d active=true \
      -d limit=10
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)

    await client.outcomes.list()
    ```

    ```python
    from protocol_api import ApiClient

    client = ApiClient(token)

    client.outcomes.list()
    ```

    ```php
    $client = new \Protocol\ApiClient($token);

    $client->outcomes->list();
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
    {
      "has_more": false,
      "data": [
        {
          "outcomes_id": "WAz8eIbvDR60rouK",
          "description": "Hello World",
        },
        {
          "outcomes_id": "hSIhXBhNe8X1d8Et"
          // ...
        }
      ]
    }
    ```

  </Col>
</Row>

## Create a outcome {{ tag: 'POST', label: '/v1/outcomes' }}

<Row>
  <Col>

    This endpoint allows you to add a new outcome to your outcome list in Protocol. To add a outcome, you must provide their Protocol description.

    ### Required attributes

    <Properties>
      <Property name="description" type="string">
        The description for the outcome.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/v1/outcomes">

    ```bash {{ title: 'cURL' }}
    curl https://api.protocol.chat/v1/outcomes \
      -H "Authorization: Bearer {token}" \
      -d description="Hello World" \
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)

    await client.outcomes.create({
      description: 'Hello World',
    })
    ```

    ```python
    from protocol_api import ApiClient

    client = ApiClient(token)

    client.outcomes.create(
      description="Hello World",
    )
    ```

    ```php
    $client = new \Protocol\ApiClient($token);

    $client->outcomes->create([
      'description' => 'Hello World',
    ]);
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
    {
       "outcomes_id": "WAz8eIbvDR60rouK",
        "description": "Hello World",
    }
    ```

  </Col>
</Row>

## Update a outcome {{ tag: 'PUT', label: '/v1/outcomes/:outcomes_id' }}

<Row>
  <Col>

    This endpoint allows you to perform an update on a outcome. Currently, the only attribute that can be updated on outcomes is the `description` attribute which controls how a outcome appears in your outcome list in Protocol.

    ### Required attributes

    <Properties>
      <Property name="description" type="string">
        The outcome display name in the outcome list.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="PUT" label="/v1/outcomes/WAz8eIbvDR60rouK">

    ```bash {{ title: 'cURL' }}
    curl -X PUT https://api.protocol.chat/v1/outcomes/WAz8eIbvDR60rouK \
      -H "Authorization: Bearer {token}" \
      -d description="Hello World"
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)

    await client.outcomes.update('WAz8eIbvDR60rouK', {
        description: 'Hello World',
    })
    ```

    ```python
    from protocol_api import ApiClient

    client = ApiClient(token)

    client.outcomes.update("WAz8eIbvDR60rouK", description="Hello World")
    ```

    ```php
    $client = new \Protocol\ApiClient($token);

    $client->outcomes->update('WAz8eIbvDR60rouK', [
      'description' => 'Hello World',
    ]);
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
    {
      "outcomes_id": "WAz8eIbvDR60rouK",
      "description": "Hello World",
    }
    ```

  </Col>
</Row>
