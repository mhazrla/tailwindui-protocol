# Course-Requirement

As the name suggests, course-requirement are a core part of Protocol. On this page, we'll dive into the different outcome endpoints you can use to manage requirement programmatically. We'll look at how to query, create, update, and delete requirement.

## The Course-requirement model

The Course-requirement model contains all the information about course-requirement, such as course and requirement information.

### Properties

<Properties>
  <Property name="course_req_id" type="string">
    Unique identifier for the requirement.
  </Property>
  <Property name="req_id" type="string">
    Unique identifier for the requirement associated with the course.
  </Property>
  <Property name="course_id" type="string">
    Unique identifier for the course associated with the outcome.
  </Property>
</Properties>

---

## List all requirement {{ tag: 'GET', label: '/v1/course-outcome' }}

<Row>
  <Col>

    This endpoint allows you to retrieve a paginated list of all your requirement. By default, a maximum of ten requirement are shown per page.

    ### Optional attributes

    <Properties>
      <Property name="limit" type="integer">
        Limit the number of requirement returned.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/v1/requirement">

    ```bash {{ title: 'cURL' }}
    curl -G https://api.protocol.chat/v1/requirement \
      -H "Authorization: Bearer {token}" \
      -d active=true \
      -d limit=10
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)

    await client.requirement.list()
    ```

    ```python
    from protocol_api import ApiClient

    client = ApiClient(token)

    client.requirement.list()
    ```

    ```php
    $client = new \Protocol\ApiClient($token);

    $client->requirement->list();
    ```

## Create a outcome {{ tag: 'POST', label: '/v1/course-outcome' }}

<Row>
  <Col>

    This endpoint allows you to add a new course-outcome list in Protocol. To add a outcome, you must provide their Protocol course_id and req_id.

    ### Required attributes

    <Properties>
      <Property name="req_id" type="string">
        Unique identifier for the requirement associated with the course.
      </Property>
      <Property name="course_id" type="string">
        Unique identifier for the course associated with the outcome.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/v1/requirement">

    ```bash {{ title: 'cURL' }}
    curl https://api.protocol.chat/v1/requirement \
      -H "Authorization: Bearer {token}" \
      -d course_id="2cvj8*Wn" \
      -d req_id="mis&2fKdqdcX" \
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)

    await client.requirement.create({
      course_id: "2cvj8*Wn",
      req_id: "mis&2fKdqdcX"
    })
    ```

    ```python
    from protocol_api import ApiClient

    client = ApiClient(token)

    client.requirement.create(
      course_id= "2cvj8*Wn",
      req_id= "mis&2fKdqdcX"
    )
    ```

    ```php

    $client->requirement->create([
      'course_id' => "2cvj8*Wn",
      'req_id' => "mis&2fKdqdcX"
    ```

## Update a outcome {{ tag: 'PUT', label: '/v1/requirement/:course_req_id' }}

<Row>
  <Col>

    This endpoint allows you to perform an update on a outcome. Currently, the only attribute that can be updated on requirement is the `req_id` and `course_id` attribute which controls how a outcome appears in your course-outcome list in Protocol.

    ### Required attributes

    <Properties>
    <Property name="req_id" type="string">
        Unique identifier for the requirement associated with the course.
      </Property>
      <Property name="course_id" type="string">
        Unique identifier for the course associated with the outcome.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="PUT" label="/v1/requirement/WAz8eIbvDR60rouK">

    ```bash {{ title: 'cURL' }}
    curl -X PUT https://api.protocol.chat/v1/requirement/WAz8eIbvDR60rouK \
      -H "Authorization: Bearer {token}" \
        -d course_id="2cvj8*Wn" \
        -d req_id="mis&2fKdqdcX" \
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)
    ```
